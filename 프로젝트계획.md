프로젝트 개요:
- 이름: StockMate (스톡메이트)
- 목적: 모의투자와 커뮤니티를 통해 안전하게 주식 투자를 연습하고 정보를 공유하는 소셜 플랫폼
- 기술: Next.js, TypeScript, PostgreSQL, Prisma, Tailwind CSS, KIS API
- 주요 기능:
  1. 실시간 주가 연동 모의투자
  2. 거래 내역 자동 기록 및 분석 (투자일지)
  3. 수익률 기반 랭킹 시스템
  4. 투자 기록 인증 및 소통 (커뮤니티)
  5. 사용자 프로필 및 팔로우

기술 스택:
- Next.js 14+ (App Router)
- React 18+
- TypeScript
- PostgreSQL (Database)
- Prisma (ORM)
- Tailwind CSS (Styling)
- shadcn/ui (UI Components)
- NextAuth.js (Authentication)
- Lightweight Charts (TradingView)
- Axios (API Communication)
- Cheerio (News Crawling)
- OpenAI API (AI Feedback)
- KIS Developers API (Stock Data)
- Jest & React Testing Library (Testing)
- Prettier & ESLint (Code Quality)

개발 방법론:
CRITICAL:   이 프로젝트는 반드시 Test-Driven Devolopment (TDD)로 개발합니다.

TDD 필수 규칙 (위반 금지):
❌ 테스트 없이 구현 코드를 먼저 작성하면 안 됩니다
❌ 테스트 실패를 확인하지 않고 구현하면 안 됩니다
❌ 구현 중에 테스트를 수정하면 안 됩니다

✅ 반드시 이 순서를 따라야 합니다:
1.  🔴 RED: 테스트를 먼저 작성 (구현은 없음)
2.  🔴 RED: 테스트 실행하여 실패 확인
3.  🟢 GREEN: 테스트를 통과시키는 최소한의 구현
4.  🟢 GREEN: 테스트 실행하여 통과 확인
5.  🔵 REFACTOR: 코드 개선 (테스트는 계속 통과)
6.  🟢 GREEN: 테스트 재실행하여 여전히 통과 확인

기능 요청을 받으면:
1.  먼저 질문: "테스트를 먼저 작성할까?"
2.  구현 전에 테스트를 먼저 작성
3.  구현 전에 테스트 실패를 확인
4.  테스트를 통과시키는 최소한의 구현만 수행
5.  구현 중에는 테스트를 절대 수정하지 않음

테스트 프레임워크:
- Jest (단위 테스트)
- 테스트 위치: tests/ 폴더
- 커버리지 목표: 80% 이상
